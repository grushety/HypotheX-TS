## SEG-023 Remote-sensing semantic pack (phenology, disturbance, InSAR)

Third semantic pack on top of SEG-021's framework: 14 labels — `greenup, senescence, peak_of_season, dormancy, disturbance, recovery, wet_up, dry_down, heatwave, coldwave, cloud_gap, APS, seasonal_deformation, linear_deformation`.  `__init__.py` gets a one-line `from . import detectors_remote_sensing` (alphabetically between hydrology and seismo-geodesy).  **Delegation pattern continues**: `bfast_breakpoint` lazily imports `fit_bfast` (reads `coefficients["breakpoint"]`, `level_left`, `level_right`); `landtrendr_positive_slope_post_disturbance` lazily imports `fit_landtrendr` (reads `coefficients["slope_2"]` — the legacy 2-segment schema; falls back to `slopes[-1]` for future generalised LandTrendr or to OLS slope when the fitter is unavailable); `etm_linear_rate` lazily imports `fit_etm` (reads `coefficients["linear_rate"]`).  The `seasonal_deformation` AC mentions "MSTL annual component" but the implementation uses a cheap FFT dominant-period check rather than fitting MSTL per segment — fitting MSTL would re-decompose what the parent shape classifier already established; the 365 ± 30 day band is enforced via the YAML `is_annual_period` predicate.  **Critical Jönsson 2004 §2 nuance**: SOS/EOS thresholds are defined relative to the *full season's* min-to-max range, not the segment's local range — using a segment-local range would mean every monotonic segment trivially crosses any threshold (a rising linspace from 0.1 to 0.6 has segment_min=0.1, segment_max=0.6, so a 20% threshold = 0.2 sits inside the segment by construction; the test would always pass).  The `_annual_min` / `_annual_max` / `_annual_amplitude` helpers therefore *prefer* caller-supplied values from context (`annual_min`, `annual_max`) over segment-local stats — documented in `_annual_amplitude`'s docstring.  Tests pass `annual_min=0.1, annual_max=0.6` in context to exercise the full-season pathway.  **Recovery test slope gotcha**: my first version used `np.linspace(0.2, 0.6, 50)` — slope = 0.4/49 ≈ 0.008, BELOW the default `min_recovery_slope=0.05`.  Fixed by using `np.linspace(0.2, 1.2, 10)` so slope ≈ 0.111 > 0.05; the tests now distinguish "passes recovery threshold" vs "below recovery threshold" cleanly.  **Same callable serves multiple labels** continues from SEG-021/022: `timesat_threshold` for greenup+senescence (predicate disambiguates by sign), `slope_plus_context` for wet_up+dry_down, `percentile_threshold_plus_duration` for heatwave+coldwave.  **`cloud_gap` is metadata-only**: reads `context['is_cloud_gap']` from the data loader's missingness mask, NEVER touches `X_seg` values — explicit test `test_cloud_gap_does_not_call_interpolation` passes a NaN-filled segment and verifies no NaN propagation.  62 tests in `test_semantic_pack_remote_sensing.py`; full backend suite at 1597 pass; ruff clean. STA/LTA peaks AT the onset, not far past it — the corrected test compares `ratio[100:110].max()` against the pre-onset baseline `ratio[60:90].mean()`.  This is the canonical Allen-1978 picker behaviour: it triggers ON the onset, then the LTA catches up and the ratio relaxes.  **Context-mutation gotcha**: my first unwrap-error test asserted `ctx['snap_2pi_multiple_count'] == 1` after calling `match_semantic_label(...)` — failed because SEG-021's matcher *copies* the user's context dict before passing it to the detector (so detector mutations don't leak between labels in `label_segment`).  Fixed by calling the detector directly when verifying context mutation, and only using `match_semantic_label` to verify the `(matched, confidence)` tuple.  Worth remembering for any future test that wants to inspect detector-augmented context: go through `DETECTOR_REGISTRY[name]` directly, not the matcher.  **`phase_jump_2pi_detector` k=0 carve-out**: snap-to-2π would return `k=0, within_tol=True` for any flat segment (zero step magnitude is trivially within 0.1π of 0·2π), so the detector explicitly rejects `k=0` to avoid tagging every flat segment as an unwrap error — covered by `test_unwrapping_error_zero_step_rejected_even_if_within_tolerance`.  **`common_mode_pca` multi-station guard**: returns `(False, 0.0)` and sets `not_applicable=True` when the caller did not supply `station_residuals` or supplied fewer than 2 rows — explicitly required by the AC.  **`obspy` is NOT a dependency**: docstring promises an optional fallback, but the module never imports obspy; scipy's `hilbert` is used (already in requirements) with absolute-value fallback if scipy is unavailable.  Test `test_sta_lta_works_without_obspy` confirms.  67 tests in `test_semantic_pack_seismo_geodesy.py`; 1524 backend pass (the same 2 pre-existing unrelated failures untouched). ## SEG-021 Hydrology semantic pack (baseflow, storm, ENSO, PDO)

Created `backend/app/services/semantic_packs/` with `core.py` (frozen `SemanticPack`/`SemanticLabel` dataclasses, `DETECTOR_REGISTRY` populated by `@register_detector` decorator, AST-validated `evaluate_predicate`, YAML `load_pack`, `match_semantic_label`, `label_segment`), `detectors_hydrology.py` (9 detectors), and `hydrology.yaml` (9 labels — `baseflow / stormflow / peak_flow / rising_limb / recession_limb / snowmelt_freshet / drought / ENSO_phase / PDO_phase`).  Each label is a `(shape_primitive, detector, context_predicate)` triple per *Formal Definitions* §2.3.  **Detector contract**: `(X_seg, shape_label, context) → (matched: bool, confidence: float)`; detectors *mutate* the `context` dict in place to add the metrics that the matching predicate references (e.g. `Q_mean`, `peak_Q`, `slope`, `dominant_period_years`).  The matcher then evaluates the YAML predicate against the augmented context.  The `baseflow` detector delegates to the SEG-016 Eckhardt fitter (`fit_eckhardt`) rather than re-implementing the recursive filter.  `slope_sign_plus_context` is the *same* detector for `rising_limb` and `recession_limb` — the predicate distinguishes by sign-of-slope and neighbour flags (`preceded_by_baseflow`, `follows_peak_flow`); the caller is expected to populate those flags from segment-context (since one segment cannot infer them from `X_seg` alone).  **Predicate evaluator security**: `evaluate_predicate` uses Python `eval` on a restricted namespace (`__builtins__: {}` + a fixed allow-list of `abs / max / min / len / all / any / round / sum`) gated by an AST whitelist (BoolOp, Compare, BinOp, UnaryOp, Constant, Name, Call, List, Tuple — explicitly excludes Attribute, Subscript, Lambda, Comprehensions, IfExp, Set/Dict literals, Starred, walrus, `__import__`).  Tests probe `__import__('os')`, attribute access, lambdas — all rejected.  Trust model documented in the docstring: pack YAML is *author-controlled* (shipped with the codebase), so the AST guard is defence-in-depth — if the trust boundary ever widens to user-uploaded packs (UI-014 import), drop `ast.Pow` from the whitelist and add a wall-clock timeout (else `2 ** 10 ** 8` would happily run).  **Auto-import gotcha**: `__init__.py` imports `detectors_hydrology` to populate `DETECTOR_REGISTRY` *before* `load_pack` runs — without this auto-import, every `load_pack("hydrology")` would raise `unknown detector` because the registry would be empty.  The `peak_flow` detector's `duration_samples` initially used `len(X_seg)` and the YAML predicate `duration_samples < 3 * dt` rejected every realistic spike fixture (segments are typically padded with baseline samples either side); fixed to count *width above half-peak height* (FWHM-style), so an 8-sample segment with one big outlier gets `duration_samples = 1`.  User-defined labels shadow pack labels by replacing entries in `pack.semantic_labels` (the dict is intentionally mutable even though `SemanticPack` itself is frozen — this is the documented per-project shadowing path from *Implementation Plan* §8.4); covered by `test_user_defined_labels_can_shadow_pack_labels`.  44 tests in `test_semantic_packs.py`; 1443 backend pass (the same 2 pre-existing unrelated failures untouched).  `label_segment` returns matches sorted by descending confidence so the UI can show the top label first.

## SEG-017 LandTrendr piecewise-linear fitter (EO trajectories)

Replaced the LandTrendr stub at `backend/app/services/decomposition/fitters/landtrendr.py` with a full Kennedy 2010 implementation.  Pipeline: (1) `find_candidate_vertices` — start from endpoints, iteratively add the time index whose residual against the *current* anchored fit is largest (greedy, Kennedy 2010 §2.2); (2) `_prune_iteratively` — drop the interior vertex whose removal least increases SSE, recording every (k, vx, sse) snapshot; (3) penalised pick — among truncations with k ≤ `max_vertices`, choose minimal `SSE + penalty_per_vertex · k`.  **Critical algorithmic gotcha**: an unanchored joint OLS over all vertex Y values puts the largest residual at the endpoints (the OLS line is mean-centred — boundary samples are far from it), so the candidate generator picks endpoint-NEIGHBOUR samples instead of true interior knots — got `[0, 5, 6, 239]` instead of `[0, 100, 200, 239]` on a clean 3-knot test fixture.  Switched to **anchored OLS** per Kennedy 2010 §2.4: endpoint vy values are *fixed* to the observed boundary samples, only interior vy values are OLS-fit.  This makes endpoint residuals exactly 0 by construction, so candidate generation picks the right knots.  Implemented in `_fit_vertices` by computing the boundary contribution `vy_left·A[:,0] + vy_right·A[:,-1]` and running `lstsq` only on the interior columns of the tent-basis design matrix.  Two further deviations from the ticket's pseudocode: (a) iterative pruning instead of `O(2^max_candidates)` brute-force `combinations()` — Bai-Perron / Kennedy 2018 EE pattern, much faster, equivalent in practice; (b) **legacy schema dual emission** for OP-021 backward compat — the new fitter emits both the generalised schema (`vertices`, `slopes`, `intercepts`, `recovery` as lists) AND the legacy 2-segment schema (`slope_1`, `slope_2`, `intercept_1`, `intercept_2`, `breakpoint`) derived from the first/last segments + first internal vertex.  This keeps the existing `tier2/trend.py` (47 OP-021 tests, all passing) working unchanged on real fitter output.  Recovery flag (`recovery[i] = True`) fires when segment `i` has positive slope, segment `i-1` has negative slope, AND `|delta_y_{i-1}| > recovery_threshold` (NDVI default 0.25).  42 tests in `test_landtrendr_fitter.py`; 1441 backend pass (the same 2 pre-existing unrelated failures untouched).  Empty-input and single-point blob constructors are explicit (`_empty_blob`, `_single_point_blob`).  Dispatcher entry `('trend', 'remote-sensing') → LandTrendr` is already in `_DISPATCH_TABLE` (set by SEG-019).

## SEG-016 Eckhardt baseflow fitter (hydrology)

Replaced `backend/app/services/decomposition/fitters/eckhardt.py` stub with the full Eckhardt 2005 two-parameter recursive baseflow filter. Public surface: `eckhardt_baseflow(Q, bfi_max=0.80, a=0.98) → DecompositionBlob` (registered as `@register_fitter("Eckhardt")`), plus `calibrate_eckhardt(Q, recession_segments) → (a, BFImax)` and the two helpers `estimate_recession_constant` (log-slope fit on user-flagged recession segments, Tallaksen 1995 §3) and `estimate_long_term_bfi` (Lyne-Hollick 1979 single-pass forward filter as a BFImax pre-estimator). Components: `{baseflow, quickflow, residual}` with residual all-zero — `Q ≡ b + q` exactly by construction (`rmse=0.0` in metadata). Coefficients: `{BFImax, a}` for OP-020 `raise_lower` to read/edit. **Critical algebraic gotcha**: the Eckhardt steady state is `b_ss = BFImax · Q`, **not** `b_ss = Q` — solving Eq. 6 with constant Q gives `b·(1−a) = (1−a)·BFImax·Q`. The first round of the constant-flow test asserted `b → Q` and failed at `b → BFImax·Q = 4` (with BFImax=0.8, Q=5); the test now asserts the correct steady state algebraically and includes a separate `BFImax→1` test to verify the `b → Q` collapse. The storm-pulse AC fixture (`base=2`, `peak=10`) uses `BFImax=0.99` so the recovered baseflow tail lies within 5 % of the underlying `b=2` truth — a lower BFImax would settle at `0.8·2=1.6`, which is correct Eckhardt behaviour but does not satisfy the AC's "recovered to 2 within 5 %" intent. Bit-identical Eq. 6 is verified in `test_eckhardt_matches_paper_equation_6_bit_identical` against an inline reference loop typed verbatim from the paper. Validation: rejects negative Q, `a` ∉ (0,1), `bfi_max` ∉ (0,1), and genuine multivariate input (2-D single-column is flattened). One caller ripple: `backend/app/services/operations/tier1/stochastic.py::_fill_baseflow` was updated from `fit_eckhardt(alpha=, bfi_max=)` (the old stub's name) to `eckhardt_baseflow(a=, bfi_max=)`; coefficient keys also renamed `{alpha, bfi_max}` → `{BFImax, a}` to match paper notation. 35 tests in `test_eckhardt_fitter.py`; full backend suite at 1434 pass (the same 2 pre-existing unrelated failures untouched). The dispatcher entry `('plateau', 'hydrology') → Eckhardt` is **not** wired in `_DISPATCH_TABLE` yet — that belongs to SEG-019 / SEG-021.

## SEG-015 BFAST decomposition fitter (EO breakpoints)

Replaced the placeholder `backend/app/services/decomposition/fitters/bfast.py` with a full BFAST fitter (Verbesselt 2010 classical + Masiliūnas 2021 lite). Uses **STL-compatible component keys** (`trend`, `seasonal`, `residual`) so all existing Tier-2 cycle/trend ops are compatible. Key design choices: (1) seasonal component is **harmonic regression** (sum of K cos/sin pairs, K = `min(3, period//2)` classical / `1` lite) rather than literal indicator dummies — smoother and better conditioned on short series, matching Verbesselt 2010 §2.1; (2) breakpoint detection uses **Chow F-test + Bai-Perron binary segmentation** rather than the literal OLS-MOSUM scan — equivalent in theory (Bai-Perron 2003 cite Chu-Hornik-Kuan 1995 for the critical-value framework) and simpler to implement; (3) the F-test in `_best_single_break` has **two scale-aware numerical floors** to handle degenerate variance: `x_var < 1e-12` rejects flat segments (was breaking the smooth-series test by detecting noise breaks after the harmonic seasonal nearly-perfectly fit the input), and `rss_full <= 1e-9 * x_var * n` rejects pure-linear inputs (was hitting div-by-zero in the F-test); a third floor `best_rss <= 1e-9 * rss_full` catches the clean-step case where the piecewise fit is exact. (4) Iterative loop converges when both `Tt` and `St` stabilise to `atol=1e-6`; non-convergence logs a warning and returns best-so-far. (5) Lite variant fixes `max_iter=1`, `n_harmonics=1`, `max_breaks=1` — single-pass with at most one breakpoint, per Masiliūnas 2021. Coefficients carry editable `break_epochs` (Python ints) and `break_magnitudes` (Python floats, jump = T(bp) − T(bp−1)) for OP-022 step ops to read and modify directly. Multivariate input is explicitly rejected; `(n,1)` 2-D shapes are flattened. 33 tests in `test_bfast_fitter.py`; 1432 backend tests pass overall (2 pre-existing unrelated failures untouched). The dispatcher already routes BFAST via `@register_fitter('BFAST')` decorator from auto-import — no changes to `dispatcher.py` were needed; SEG-019's domain-hint dispatch table already contains the entries that need to point at BFAST when remote-sensing semantics are wired up.

## OP-013 Tier-1 stochastic atoms (suppress / add_uncertainty)

`suppress` and `add_uncertainty` in `backend/app/services/operations/tier1/stochastic.py`. `suppress` supports five fill strategies: `linear` (linspace between context means), `spline` (CubicSpline on ctx_pre/ctx_post anchors), `stl_trend` (delegates to statsmodels STL, requires `aux['period']`; falls back to linear with WARNING when series too short), `climatology` (DOY-lookup from dict or array; raises ValueError with context on missing key/OOB), `baseflow` (delegates to SEG-016 Eckhardt fitter). Domain-hint defaults: `remote_sensing → climatology`, `hydrology → baseflow`, else `linear`. `add_uncertainty` injects white/pink/red colored noise (Timmer & König 1995) via `colorednoise.powerlaw_psd_gaussian`, scaled to σ by std-normalization; seed-reproducible. Both return `StochasticOpResult(frozen=True, eq=False)`. Relabeling: `suppress → RECLASSIFY_VIA_SEGMENTER`, `add_uncertainty → PRESERVED`. AuditEvent emission deferred to OP-041 (same gap as OP-010/011/012). New dep: `colorednoise>=0.2`. 53 tests in `test_stochastic_ops.py`.

## SEG-014 STL/MSTL decomposition fitter (Cleveland 1990 / Bandara 2021)

Replaced stubs in `backend/app/services/decomposition/fitters/stl.py` and `mstl.py` with full implementations via statsmodels (requires `statsmodels>=0.14` in requirements.txt). `detect_dominant_period(X)` in `stl.py` returns `int` (single period) or `list[int]` (multiple) via FFT + ACF. Two non-obvious gotchas: (1) statsmodels MSTL sorts periods internally, so `valid_periods` must be sorted before naming `seasonal_{T}` columns or you get mislabeled components; (2) MSTL crashes with `UnboundLocalError` when `2*period >= n` for all requested periods — handled by an underdetermined fallback in `_fit_mstl_1d`. Dispatches: `("cycle", None)` → STL, `("cycle", "multi-period")` → MSTL. Tests: `backend/tests/test_stl_mstl_fitter.py` (37 tests).

## SEG-013 ETM decomposition fitter (Bevis-Brown 2014)

Replaced the linear-trend stub in `backend/app/services/decomposition/fitters/etm.py` with the full Bevis-Brown (2014) Eq. 1 ETM model: x₀ + linear rate + Heaviside steps + log/exp transients + sinusoidal harmonics, fitted via `np.linalg.lstsq`. `build_etm_design_matrix(t, known_steps, known_transients, harmonic_periods)` returns `(A, labels)` where labels follow Bevis-Brown naming (`x0`, `linear_rate`, `step_at_{t_s}`, `log_{t_r}_tau{τ}`, `sin_{T}`, etc.). Graceful underdetermined fallback (n < p) returns a constant-mean blob with `fit_metadata["underdetermined"] = True`. Multivariate (n, d) input fits channels independently and stacks. Registered as `"ETM"` and dispatched for `trend`, `step`, and `transient` shapes. Tests: `backend/tests/test_etm_fitter.py` (29 tests).

## OP-012 replace_from_library + DonorEngine (3 backends)

`replace_from_library`, `NativeGuide`, `SETSDonor`, `DiscordDonor`, `DonorEngine` Protocol in `backend/app/services/operations/tier1/replace_from_library.py`. Crossfade via `w = linspace(0,1,cw)` at boundaries ensures `result[0] == X_seg[0]` and `result[-1] == X_seg[-1]`. Length mismatch handled by `np.interp` before crossfade. **NativeGuide** uses `tslearn.metrics.dtw` for nearest-unlike-neighbour. **SETSDonor** is a vendored implementation of Bahri et al. (2022) Algorithm 1: medoid of z-normalised sliding-window subsequences tiled to target length (omarbahri/SETS is unmaintained). **DiscordDonor** uses `stumpy.stump` and returns the subsequence at `argmax(mp[:,0])` (no target_class filtering — follows the paper pseudocode; callers should pass a class-filtered corpus). Relabeling is always RECLASSIFY_VIA_SEGMENTER. New deps: `tslearn>=0.6`, `stumpy>=1.12` in requirements.txt. UI-008 integration test deferred to UI-008.

## OP-011 Tier-1 time atoms (time_shift / reverse_time / resample)

`time_shift`, `reverse_time`, `resample` in `backend/app/services/operations/tier1/time.py`. All return `TimeOpResult(values, op_name, relabel=PRESERVED, tier=1)` — time-axis manipulation never changes shape. `time_shift(delta_t, taper_width=5)` uses `np.roll` + a linear taper on the wrap-around edge to avoid discontinuities. `reverse_time` is `arr[::-1].copy()`. `resample` has three back-ends: `antialiased` (scipy `decimate` with FIR LPF before downsampling; `resample_poly` for upsampling — Oppenheim Ch. 4), `sg` (SG smooth + `np.interp` — Savitzky-Golay 1964), `linear` (raw `np.interp`). SG window adapts to signal length. Used as primitives by OP-022/023/024/025. Exported from `tier1/__init__.py`.

## SEG-020 Few-shot support buffer with drift tracking

`SupportBuffer` added at `backend/app/services/suggestion/support_buffer.py`. Wraps all 7-shape-class `deque(maxlen=cap_per_class)` buffers and sits between `BoundarySuggestionService.accept_correction()` and `PrototypeShapeClassifier.fit_prototypes()`. `accept_correction(segment_X, label, confidence, classifier)` gates on `confidence_gate` (default 0.7) and valid SHAPE_LABELS; rejected corrections do not advance `total_accepted`. ## SEG-011 Prototype encoder + classifier (few-shot, real corrections only)

Evaluation fixture in `test_prototype_classifier.py` uses spike signals of length 30 (> `spike_max_len=20` rule threshold) to guarantee the prototype classifier outperforms the rule classifier by ≥ 3 pts macro F1 — the rule classifier misclassifies all long spikes while the prototype classifier embeds them correctly.

## OP-023 Spike Tier-2 ops (6 ops)

Created `backend/app/services/operations/tier2/spike.py` with `remove`, `clip_cap`, `amplify`, `smear_to_transient`, `duplicate`, `shift_time`. Spike ops work on raw signal values (not decomposition coefficients) since spikes are residual outliers with no parametric blob. Key gotchas: (1) `amplify` uses a multiplicative Gaussian envelope (`arr * (1 + w * (alpha-1))`), so a zero-valued baseline produces zero diff — tests must use a non-zero baseline signal; (2) `smear_to_transient` uses a manually-implemented Ricker wavelet (scipy.signal.ricker was removed in scipy >= 1.12) with L1 normalization; (3) `shift_time` caps `taper_width=max(1, min(5, n-1))` to handle near-single-sample spikes (n < 6) without raising a tier1 ValueError; (4) `remove(method='chen_sg')` is an iterative SG-envelope fitting that converges after n_iter=3 by default. AuditEvent deferred to OP-041. 63 tests in `test_spike_ops.py`.

## OP-022 Step Tier-2 ops (6 ops)

Created `backend/app/services/operations/tier2/step.py` with `de_jump`, `invert_sign`, `scale_magnitude`, `shift_in_time`, `convert_to_ramp`, `duplicate`. All ops work on ETM Heaviside coefficients using key `step_at_{t_s:.6g}`; no raw-signal manipulation. Key gotchas: (1) `de_jump` and `scale_magnitude(alpha=0)` produce bit-identical values — both guaranteed since both zero the step coefficient and component via the same code path; (2) `convert_to_ramp` replaces the Heaviside key with `log_{t_s:.6g}_tau{tau_ramp:.6g}` and emits DETERMINISTIC('transient'); (3) `duplicate` raises `ValueError` when `delta_t=0` to prevent silent overwrite of the original step; (4) `shift_in_time` pops the old key and inserts a new one, preserving amplitude. ## OP-026 Noise Tier-2 ops (5 ops)

Created `backend/app/services/operations/tier2/noise.py` with `suppress_denoise`, `amplify`, `change_color`, `inject_synthetic`, `whiten`. Also created `backend/app/services/noise_models/` with `NoiseModel` Protocol (runtime_checkable) + `AR1NoiseModel`, `FlickerNoiseModel`, `GammaSpeckleModel`. Key design: `suppress_denoise` takes raw ndarray (like spike ops, shape-agnostic); the other 4 take `blob` and operate on `components['residual']` to preserve trend/cycle structure. `suppress_denoise` supports BayesShrink (sym8 wavelet, MAD threshold, Chang 2000), SG, TV (Chambolle/scikit-image), Kalman-RTS smoother, GACOS. Key gotchas: (1) Kalman smoother needs `q>0` and `r>0` guards — `q=0` drives `p_filt→0` then `0/0=NaN`; (2) `whiten` needs isfinite guard — near-constant residual gives PSD≈0 → whitening→Inf → NaN via IEEE `Inf×0`; (3) `change_color` normalises new noise to original residual σ (not signal σ) to preserve trend/cycle amplitude. New deps: `PyWavelets>=1.4`, `scikit-image>=0.22`. 74 tests in `test_noise_ops.py`.

## OP-025 Transient Tier-2 ops (9 ops)

Created `backend/app/services/operations/tier2/transient.py` with `remove`, `amplify`, `dampen`, `shift_time`, `change_duration`, `change_decay_constant`, `replace_shape`, `duplicate`, `convert_to_step`. Dispatches by `blob.method` ('ETM' or 'GrAtSiD'). Key gotchas: (1) `remove` and `amplify` on GrAtSiD full-features (`coefficients['features']` list) require a `t` parameter — without it the component array would be stale after coefficient mutation; (2) all GrAtSiD full-features ops recompute `components['transient']` as the sum of ALL features after modifying one (not just the modified feature's waveform); (3) `replace_shape` GrAtSiD uses a unit-basis-vector OLS refit (amplitude=1 dummy feature) to avoid NaN/Inf when amplitude=0; (4) `convert_to_step` GrAtSiD uses `pop(int(feature_id))` not `list.remove()` to avoid value-equality collision on duplicate features; (5) `replace_shape` rejects `new_basis='both'` for GrAtSiD (single-type feature schema); (6) `duplicate` raises ValueError when `delta_t=0`; (7) `dampen` validates alpha∈(0,1]. ETM feature_id = key string like `'log_60_tau20'`; GrAtSiD full feature_id = int index; GrAtSiD stub feature_id = `'transient'`.
