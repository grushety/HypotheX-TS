## OP-021 Trend Tier-2 ops (6 ops)

Created `backend/app/services/operations/tier2/trend.py` with `flatten`, `change_slope`, `reverse_direction`, `linearise`, `extrapolate`, `add_acceleration`. Dispatches by `blob.method` ('ETM' or 'LandTrendr'). Key gotchas: (1) `flatten` delegates to `change_slope(alpha=0)` — both guaranteed identical; (2) `linearise` uses Theil-Sen (Sen 1968) and stores residual in `blob.residual` only (not in components) so `reassemble()` = fitted line, not original signal; (3) `extrapolate` uses absolute-t convention `x0 + rate*t_ext`, not `t_ext-t_ext[0]` — the reviewer caught a bug here that masked in zero-start tests; (4) LandTrendr flatten/change_slope(0) collapses to Constant method to avoid intercept_1≠intercept_2 step artifact. Relabeling: flatten/change_slope(0) → DETERMINISTIC('plateau'); others → PRESERVED('trend'). AuditEvent deferred to OP-041. 47 tests.

## OP-020 Plateau Tier-2 ops (5 ops)

Created `backend/app/services/operations/tier2/` package and `plateau.py` with `raise_lower`, `invert`, `replace_with_trend`, `replace_with_cycle`, `tilt_detrend`. All ops return `Tier2OpResult(frozen=True, eq=False, tier=2)`. The three mutating ops (`raise_lower`, `replace_with_trend`, `replace_with_cycle`) deepcopy the blob internally — callers never need to deepcopy. `replace_with_trend` transitions blob to `ETM` method (x0 + linear_rate components, Bevis & Brown 2014 Eq. 1); `replace_with_cycle` transitions to `STL` method (trend + seasonal + residual, Cleveland et al. 1990). Relabeling: PRESERVED for raise_lower/invert/tilt_detrend; DETERMINISTIC('trend'/'cycle') for replace ops. AuditEvent deferred to OP-041; UI gating deferred to UI-006. 50 tests in `test_plateau_ops.py`.

## OP-013 Tier-1 stochastic atoms (suppress / add_uncertainty)

`suppress` and `add_uncertainty` in `backend/app/services/operations/tier1/stochastic.py`. `suppress` supports five fill strategies: `linear` (linspace between context means), `spline` (CubicSpline on ctx_pre/ctx_post anchors), `stl_trend` (delegates to statsmodels STL, requires `aux['period']`; falls back to linear with WARNING when series too short), `climatology` (DOY-lookup from dict or array; raises ValueError with context on missing key/OOB), `baseflow` (delegates to SEG-016 Eckhardt fitter). Domain-hint defaults: `remote_sensing → climatology`, `hydrology → baseflow`, else `linear`. `add_uncertainty` injects white/pink/red colored noise (Timmer & König 1995) via `colorednoise.powerlaw_psd_gaussian`, scaled to σ by std-normalization; seed-reproducible. Both return `StochasticOpResult(frozen=True, eq=False)`. Relabeling: `suppress → RECLASSIFY_VIA_SEGMENTER`, `add_uncertainty → PRESERVED`. AuditEvent emission deferred to OP-041 (same gap as OP-010/011/012). New dep: `colorednoise>=0.2`. 53 tests in `test_stochastic_ops.py`.

## SEG-014 STL/MSTL decomposition fitter (Cleveland 1990 / Bandara 2021)

Replaced stubs in `backend/app/services/decomposition/fitters/stl.py` and `mstl.py` with full implementations via statsmodels (requires `statsmodels>=0.14` in requirements.txt). `detect_dominant_period(X)` in `stl.py` returns `int` (single period) or `list[int]` (multiple) via FFT + ACF. Two non-obvious gotchas: (1) statsmodels MSTL sorts periods internally, so `valid_periods` must be sorted before naming `seasonal_{T}` columns or you get mislabeled components; (2) MSTL crashes with `UnboundLocalError` when `2*period >= n` for all requested periods — handled by an underdetermined fallback in `_fit_mstl_1d`. Dispatches: `("cycle", None)` → STL, `("cycle", "multi-period")` → MSTL. Tests: `backend/tests/test_stl_mstl_fitter.py` (37 tests).

## SEG-013 ETM decomposition fitter (Bevis-Brown 2014)

Replaced the linear-trend stub in `backend/app/services/decomposition/fitters/etm.py` with the full Bevis-Brown (2014) Eq. 1 ETM model: x₀ + linear rate + Heaviside steps + log/exp transients + sinusoidal harmonics, fitted via `np.linalg.lstsq`. `build_etm_design_matrix(t, known_steps, known_transients, harmonic_periods)` returns `(A, labels)` where labels follow Bevis-Brown naming (`x0`, `linear_rate`, `step_at_{t_s}`, `log_{t_r}_tau{τ}`, `sin_{T}`, etc.). Graceful underdetermined fallback (n < p) returns a constant-mean blob with `fit_metadata["underdetermined"] = True`. Multivariate (n, d) input fits channels independently and stacks. Registered as `"ETM"` and dispatched for `trend`, `step`, and `transient` shapes. Tests: `backend/tests/test_etm_fitter.py` (29 tests).

## OP-012 replace_from_library + DonorEngine (3 backends)

`replace_from_library`, `NativeGuide`, `SETSDonor`, `DiscordDonor`, `DonorEngine` Protocol in `backend/app/services/operations/tier1/replace_from_library.py`. Crossfade via `w = linspace(0,1,cw)` at boundaries ensures `result[0] == X_seg[0]` and `result[-1] == X_seg[-1]`. Length mismatch handled by `np.interp` before crossfade. **NativeGuide** uses `tslearn.metrics.dtw` for nearest-unlike-neighbour. **SETSDonor** is a vendored implementation of Bahri et al. (2022) Algorithm 1: medoid of z-normalised sliding-window subsequences tiled to target length (omarbahri/SETS is unmaintained). **DiscordDonor** uses `stumpy.stump` and returns the subsequence at `argmax(mp[:,0])` (no target_class filtering — follows the paper pseudocode; callers should pass a class-filtered corpus). Relabeling is always RECLASSIFY_VIA_SEGMENTER. New deps: `tslearn>=0.6`, `stumpy>=1.12` in requirements.txt. UI-008 integration test deferred to UI-008.

## OP-011 Tier-1 time atoms (time_shift / reverse_time / resample)

`time_shift`, `reverse_time`, `resample` in `backend/app/services/operations/tier1/time.py`. All return `TimeOpResult(values, op_name, relabel=PRESERVED, tier=1)` — time-axis manipulation never changes shape. `time_shift(delta_t, taper_width=5)` uses `np.roll` + a linear taper on the wrap-around edge to avoid discontinuities. `reverse_time` is `arr[::-1].copy()`. `resample` has three back-ends: `antialiased` (scipy `decimate` with FIR LPF before downsampling; `resample_poly` for upsampling — Oppenheim Ch. 4), `sg` (SG smooth + `np.interp` — Savitzky-Golay 1964), `linear` (raw `np.interp`). SG window adapts to signal length. Used as primitives by OP-022/023/024/025. Exported from `tier1/__init__.py`.

## OP-010 Tier-1 amplitude atoms (scale / offset / mute_zero)

`scale`, `offset`, `mute_zero` in `backend/app/services/operations/tier1/amplitude.py`. All return `AmplitudeOpResult(values, relabel, op_name, tier=1)`. Blob-aware dispatch by `blob.method`: Constant scales/offsets the `trend` component and `level` coefficient; ETM offsets the `x0` component/coefficient and scales `sin_*`/`cos_*` harmonic components; STL/MSTL scale/offset the `trend` (and STL `seasonal`, MSTL `seasonal_*`). Unknown blob methods fall back to raw-value arithmetic with a WARNING log. Relabeling is inline (OP-040 not yet done): `scale(α=0)` → DETERMINISTIC('plateau'); `scale(α≠0)` / `offset(δ)` → PRESERVED; `mute_zero` → DETERMINISTIC('plateau') for both fill modes. `mute_zero` replaces blob components with `{'muted': out}` and coefficients with `{'fill': ..., 'fill_value': ...}` so `reassemble()` still works. Exported from `tier1/__init__.py`.

## SEG-020 Few-shot support buffer with drift tracking

`SupportBuffer` added at `backend/app/services/suggestion/support_buffer.py`. Wraps all 7-shape-class `deque(maxlen=cap_per_class)` buffers and sits between `BoundarySuggestionService.accept_correction()` and `PrototypeShapeClassifier.fit_prototypes()`. `accept_correction(segment_X, label, confidence, classifier)` gates on `confidence_gate` (default 0.7) and valid SHAPE_LABELS; rejected corrections do not advance `total_accepted`. Every `n_update` (default 5) accepted corrections, calls `classifier.fit_prototypes(all_buffered_segments)` and computes drift as `max_y ‖μ_y^new − μ_y^old‖₂`; drift > `drift_threshold` logs a WARNING for UI-002. `prev_prototypes` is a copy of pre-update prototypes retained for rollback. `to_dict()`/`from_dict()` persists buffer segments + config but NOT prototype vectors (they are recomputed from segments after loading). Exported from `suggestion/__init__.py`. 28 tests in `test_support_buffer.py`.

## SEG-011 Prototype encoder + classifier (few-shot, real corrections only)

`PrototypeShapeClassifier` added to `backend/app/services/suggestion/prototype_classifier.py`, alongside the legacy `PrototypeChunkClassifier`. Uses the 7-shape vocabulary (`SHAPE_LABELS`), computing L2-normalized mean prototypes exclusively from `provenance='user'` support segments (synthetic/template are hard-rejected). `BoundarySuggestionService.propose()` activates it when ≥ 5 corrections per class exist across ≥ 4 of 7 shape classes; below threshold it falls back to SEG-008 rule-based. A two-way label bridge (`_DOMAIN_TO_SHAPE`, `_PRIMITIVE_TO_DOMAIN` in `suggestions.py`) handles the mismatch between the 7-shape vocab and the 6-label domain vocab (`event↔transient`, `transition↔step`, `periodic↔cycle`). The `PrototypeShapeClassifier` is DI-injected as `self._shape_classifier` in `BoundarySuggestionService.__init__`. Evaluation fixture in `test_prototype_classifier.py` uses spike signals of length 30 (> `spike_max_len=20` rule threshold) to guarantee the prototype classifier outperforms the rule classifier by ≥ 3 pts macro F1 — the rule classifier misclassifies all long spikes while the prototype classifier embeds them correctly.

## OP-024 Cycle Tier-2 ops (7 ops)

Created `backend/app/services/operations/tier2/cycle.py` with `deseasonalise_remove`, `amplify_amplitude`, `dampen_amplitude`, `phase_shift`, `change_period`, `change_harmonic_content`, `replace_with_flat`. All ops work on STL (`components['seasonal']`) and MSTL (`components['seasonal_{T}']`) blobs. Key gotchas: (1) `replace_with_flat` must zero the residual component (not just seasonal + trend) to guarantee a constant plateau — tests with `residual=zeros` mask this bug; (2) `change_period` renames MSTL keys from `seasonal_10` → `seasonal_20` etc. to keep key names in sync with the actual period; (3) `phase_shift(method='hilbert')` applies a linear-blend edge taper over `min(4, n//4)` samples to mitigate FFT boundary artifacts; `method='harmonic'` uses rfft rotation as a numerically stable fallback; (4) `dampen_amplitude` validates alpha ∈ (0, 1] — use `amplify_amplitude(0)` to fully remove seasonal; (5) `change_harmonic_content` uses OLS to extract and replace the k-th harmonic relative to each component's period T. AuditEvent deferred to OP-041. 67 tests in `test_cycle_ops.py`.

## OP-023 Spike Tier-2 ops (6 ops)

Created `backend/app/services/operations/tier2/spike.py` with `remove`, `clip_cap`, `amplify`, `smear_to_transient`, `duplicate`, `shift_time`. Spike ops work on raw signal values (not decomposition coefficients) since spikes are residual outliers with no parametric blob. Key gotchas: (1) `amplify` uses a multiplicative Gaussian envelope (`arr * (1 + w * (alpha-1))`), so a zero-valued baseline produces zero diff — tests must use a non-zero baseline signal; (2) `smear_to_transient` uses a manually-implemented Ricker wavelet (scipy.signal.ricker was removed in scipy >= 1.12) with L1 normalization; (3) `shift_time` caps `taper_width=max(1, min(5, n-1))` to handle near-single-sample spikes (n < 6) without raising a tier1 ValueError; (4) `remove(method='chen_sg')` is an iterative SG-envelope fitting that converges after n_iter=3 by default. AuditEvent deferred to OP-041. 63 tests in `test_spike_ops.py`.

## OP-022 Step Tier-2 ops (6 ops)

Created `backend/app/services/operations/tier2/step.py` with `de_jump`, `invert_sign`, `scale_magnitude`, `shift_in_time`, `convert_to_ramp`, `duplicate`. All ops work on ETM Heaviside coefficients using key `step_at_{t_s:.6g}`; no raw-signal manipulation. Key gotchas: (1) `de_jump` and `scale_magnitude(alpha=0)` produce bit-identical values — both guaranteed since both zero the step coefficient and component via the same code path; (2) `convert_to_ramp` replaces the Heaviside key with `log_{t_s:.6g}_tau{tau_ramp:.6g}` and emits DETERMINISTIC('transient'); (3) `duplicate` raises `ValueError` when `delta_t=0` to prevent silent overwrite of the original step; (4) `shift_in_time` pops the old key and inserts a new one, preserving amplitude. AuditEvent emission deferred to OP-041. 56 tests in `test_step_ops.py`.

## OP-025 Transient Tier-2 ops (9 ops)

Created `backend/app/services/operations/tier2/transient.py` with `remove`, `amplify`, `dampen`, `shift_time`, `change_duration`, `change_decay_constant`, `replace_shape`, `duplicate`, `convert_to_step`. Dispatches by `blob.method` ('ETM' or 'GrAtSiD'). Key gotchas: (1) `remove` and `amplify` on GrAtSiD full-features (`coefficients['features']` list) require a `t` parameter — without it the component array would be stale after coefficient mutation; (2) all GrAtSiD full-features ops recompute `components['transient']` as the sum of ALL features after modifying one (not just the modified feature's waveform); (3) `replace_shape` GrAtSiD uses a unit-basis-vector OLS refit (amplitude=1 dummy feature) to avoid NaN/Inf when amplitude=0; (4) `convert_to_step` GrAtSiD uses `pop(int(feature_id))` not `list.remove()` to avoid value-equality collision on duplicate features; (5) `replace_shape` rejects `new_basis='both'` for GrAtSiD (single-type feature schema); (6) `duplicate` raises ValueError when `delta_t=0`; (7) `dampen` validates alpha∈(0,1]. ETM feature_id = key string like `'log_60_tau20'`; GrAtSiD full feature_id = int index; GrAtSiD stub feature_id = `'transient'`. Relabeling: remove/duplicate→RECLASSIFY; others→PRESERVED; convert_to_step→DETERMINISTIC('step'). AuditEvent deferred to OP-041. 91 tests in `test_transient_ops.py`.
