## SEG-017 LandTrendr piecewise-linear fitter (EO trajectories)

Replaced the LandTrendr stub at `backend/app/services/decomposition/fitters/landtrendr.py` with a full Kennedy 2010 implementation.  Pipeline: (1) `find_candidate_vertices` — start from endpoints, iteratively add the time index whose residual against the *current* anchored fit is largest (greedy, Kennedy 2010 §2.2); (2) `_prune_iteratively` — drop the interior vertex whose removal least increases SSE, recording every (k, vx, sse) snapshot; (3) penalised pick — among truncations with k ≤ `max_vertices`, choose minimal `SSE + penalty_per_vertex · k`.  **Critical algorithmic gotcha**: an unanchored joint OLS over all vertex Y values puts the largest residual at the endpoints (the OLS line is mean-centred — boundary samples are far from it), so the candidate generator picks endpoint-NEIGHBOUR samples instead of true interior knots — got `[0, 5, 6, 239]` instead of `[0, 100, 200, 239]` on a clean 3-knot test fixture.  Switched to **anchored OLS** per Kennedy 2010 §2.4: endpoint vy values are *fixed* to the observed boundary samples, only interior vy values are OLS-fit.  This makes endpoint residuals exactly 0 by construction, so candidate generation picks the right knots.  Implemented in `_fit_vertices` by computing the boundary contribution `vy_left·A[:,0] + vy_right·A[:,-1]` and running `lstsq` only on the interior columns of the tent-basis design matrix.  Two further deviations from the ticket's pseudocode: (a) iterative pruning instead of `O(2^max_candidates)` brute-force `combinations()` — Bai-Perron / Kennedy 2018 EE pattern, much faster, equivalent in practice; (b) **legacy schema dual emission** for OP-021 backward compat — the new fitter emits both the generalised schema (`vertices`, `slopes`, `intercepts`, `recovery` as lists) AND the legacy 2-segment schema (`slope_1`, `slope_2`, `intercept_1`, `intercept_2`, `breakpoint`) derived from the first/last segments + first internal vertex.  This keeps the existing `tier2/trend.py` (47 OP-021 tests, all passing) working unchanged on real fitter output.  Recovery flag (`recovery[i] = True`) fires when segment `i` has positive slope, segment `i-1` has negative slope, AND `|delta_y_{i-1}| > recovery_threshold` (NDVI default 0.25).  42 tests in `test_landtrendr_fitter.py`; 1441 backend pass (the same 2 pre-existing unrelated failures untouched).  Empty-input and single-point blob constructors are explicit (`_empty_blob`, `_single_point_blob`).  Dispatcher entry `('trend', 'remote-sensing') → LandTrendr` is already in `_DISPATCH_TABLE` (set by SEG-019).

## OP-026 Noise Tier-2 ops (5 ops)
